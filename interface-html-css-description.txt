==================================================
1. С чего начинается создание визуального интерфейса
==================================================

Чтобы создать интерфейс калькулятора, обычно нужно пройти несколько этапов:

1. Определить, какие блоки будут на странице.
2. Разметить эти блоки в HTML.
3. Связать HTML с CSS через файл стилей.
4. Задать внешний вид страницы и элементов в CSS.
5. Настроить расположение блоков, размеры, отступы, цвета, шрифты, анимации и адаптивность.
6. Подключить JavaScript, чтобы интерфейс стал интерактивным.

В этом проекте используется именно такая схема:
- index.html создает структуру страницы;
- style.css отвечает за внешний вид;
- script.js добавляет поведение.

HTML отвечает на вопрос "что находится на странице".
CSS отвечает на вопрос "как это выглядит".
JavaScript отвечает на вопрос "как это работает".

==================================================
2. Как устроен HTML в этом проекте
==================================================

Файл index.html содержит всю структуру интерфейса калькулятора.

Разберем его по частям.

2.1. Объявление типа документа

Строка:

<!DOCTYPE html>

означает, что браузер должен интерпретировать документ как современный HTML5.

Это важно, потому что без правильного DOCTYPE браузер может перейти в режим совместимости со старыми стандартами, и стили будут работать нестабильно.

2.2. Тег html

Строка:

<html lang="ru">

говорит браузеру, что:
- это корневой HTML-элемент всей страницы;
- основной язык страницы — русский.

Атрибут lang="ru" полезен для:
- поисковых систем;
- экранных дикторов;
- корректной языковой обработки текста.

2.3. Блок head

Внутри <head> находятся служебные данные страницы.

2.3.1. Кодировка

<meta charset="UTF-8" />

Это указывает кодировку UTF-8. Благодаря этому русский текст, специальные символы и математические знаки отображаются правильно.

2.3.2. Настройка адаптивности

<meta name="viewport" content="width=device-width, initial-scale=1.0" />

Это очень важная строка для мобильных устройств.

Она говорит браузеру:
- ширина области отображения должна соответствовать ширине устройства;
- начальный масштаб должен быть 1.0.

Без нее мобильная версия часто выглядит слишком мелкой или масштабируется неправильно.

2.3.3. Заголовок страницы

<title>Tech Calculator</title>

Этот текст отображается:
- во вкладке браузера;
- в названии окна;
- в закладках.

2.3.4. Подключение CSS

<link rel="stylesheet" href="style.css">

Здесь HTML подключает внешний файл стилей.

Это значит, что все CSS-правила из style.css будут применены к элементам страницы.

==================================================
3. Структура body: что именно находится на странице
==================================================

Внутри <body> размещается весь интерфейс, который видит пользователь.

3.1. Обертка страницы

<div class="wrap">

Это внешний контейнер для центровки калькулятора.

Он нужен, чтобы:
- поместить калькулятор по центру экрана;
- дать внешние отступы;
- управлять позиционированием.

3.2. Основной блок калькулятора

<section class="calculator" aria-label="Калькулятор">

Это главный визуальный блок приложения.

Почему используется именно section:
- section подходит для логически самостоятельного раздела страницы;
- калькулятор здесь является отдельным интерфейсным модулем.

Атрибут aria-label="Калькулятор" нужен для доступности:
- экранный диктор сможет понять, что это за область;
- интерфейс становится более понятным для пользователей вспомогательных технологий.

3.3. Декоративная линия сканирования

<div class="scanline"></div>

Это декоративный элемент. Он не несет содержательной информации, а нужен для стилизации под tech/futuristic-интерфейс.

3.4. Верхняя панель

<div class="topbar">

Внутри нее находятся:
- блок бренда;
- индикатор режима.

3.4.1. Блок бренда

<div class="brand">

Содержит:
- <div class="brand-badge"></div> — светящуюся точку;
- <div class="brand-text"> — текстовую часть.

Внутри brand-text:
- brand-title — главное название "Quantum Calc";
- brand-sub — подзаголовок "Минималистичный tech-калькулятор".

Это декоративно-информационная зона, которая формирует стиль интерфейса.

3.4.2. Индикатор режима

<div class="mode-pill">

В нем находятся:
- <span class="mode-dot"></span> — цветной индикатор;
- <span>ONLINE</span> — текстовый статус.

Этот блок стилизован как небольшая капсула и усиливает впечатление "технического прибора".

3.5. Экран калькулятора

<div class="display">

Это один из важнейших блоков интерфейса. Он отвечает за визуальное отображение:
- статуса;
- предыдущего выражения;
- текущего выражения или результата.

3.5.1. Верхняя часть дисплея

<div class="display-top">

Содержит:
- history-label — подпись "last operation";
- status — элемент, где JavaScript показывает текущее состояние.

Элемент status имеет id="status", потому что JavaScript обращается к нему напрямую через document.getElementById('status').

3.5.2. Предыдущее выражение

<div class="previous" id="previous"></div>

Этот блок показывает прошлое выражение, например:
- 2+2 =
- √(9)
- 5²

Он нужен, чтобы пользователь видел контекст вычисления.

3.5.3. Текущее значение

<div class="current" id="current">0</div>

Это главный экран калькулятора.

Он показывает:
- текущее набранное выражение;
- либо итоговый результат.

Начальное значение "0" задается сразу в HTML.

3.6. Блок кнопок управления

<div class="controls" id="controls">

Здесь размещены все кнопки калькулятора.

Каждая кнопка оформлена тегом <button>, потому что:
- это семантически правильный элемент для действия;
- он уже поддерживает клики, фокус, доступность и нажатия с клавиатуры.

Кнопки разделены на два типа:
- кнопки с data-value — добавляют символ в выражение;
- кнопки с data-action — вызывают действие.

Примеры:
- data-value="7" — добавить цифру 7;
- data-value="+" — добавить оператор;
- data-action="clear" — очистить все;
- data-action="calculate" — вычислить результат.

Такой подход удобен, потому что JavaScript может универсально понять, что делать при нажатии каждой кнопки.

3.7. Блок подсказок

<div class="tips">

Внутри находятся короткие подсказки:
- Enter = посчитать
- Backspace = удалить
- Shift+C = полный сброс

Это улучшает usability, то есть удобство использования.

3.8. Подключение JavaScript

В конце body подключается:

<script src="script.js"></script>

Скрипт размещен внизу страницы, чтобы HTML сначала успел загрузиться, а потом JavaScript уже получил доступ к готовым элементам.

==================================================
4. Почему классы и id в HTML так важны
==================================================

В этом проекте используются и class, и id.

4.1. class

Class нужен для:
- стилизации группы похожих элементов;
- повторного использования стилей;
- логического деления интерфейса.

Например:
- class="operator"
- class="danger"
- class="equal"
- class="wide"

Один и тот же класс можно назначить многим элементам.

4.2. id

Id нужен для уникального элемента на странице.

Здесь id используются там, где JavaScript должен получить конкретный элемент:
- id="current"
- id="previous"
- id="status"
- id="controls"

Id должен быть уникальным в пределах страницы.

==================================================
5. Как работает CSS в этом проекте
==================================================

CSS применяет стили к HTML-элементам через селекторы.

Пример общего вида CSS-правила:

селектор {
    свойство: значение;
}

Селектор указывает, к каким элементам применять стиль.
Свойство говорит, что именно менять.
Значение указывает, каким будет результат.

Пример:

.wrap {
    display: grid;
    place-items: center;
}

Здесь:
- .wrap — селектор класса;
- display — CSS-свойство;
- grid — значение свойства display;
- place-items: center — центрирование содержимого.

==================================================
6. Разбор CSS по основным блокам
==================================================

6.1. Псевдокласс :root

В начале файла есть:

:root {
    ...
}

`:root` — это корневой элемент документа. Обычно это html.

Здесь в нем объявлены CSS-переменные:
- --bg-1
- --bg-2
- --panel
- --text
- --muted
- --accent
- --danger
- --success
- --radius
- --transition
и другие.

Что такое CSS-переменные:
- это именованные значения, которые можно многократно использовать в стилях;
- они уменьшают дублирование;
- упрощают изменение темы.

Пример:

--accent: #51d6ff;

Позже это значение используется так:

color: var(--accent);

Это значит: "взять значение переменной --accent".

Почему это удобно:
- если нужно поменять акцентный цвет, достаточно изменить его один раз в :root;
- все элементы автоматически получат новый цвет.

6.2. Универсальный селектор *

* {
    box-sizing: border-box;
}

Селектор `*` выбирает все элементы на странице.

Свойство:
- box-sizing: border-box;

означает, что ширина и высота элемента будут включать:
- content;
- padding;
- border.

Это делает верстку намного предсказуемее.

Без border-box ширина элемента могла бы неожиданно увеличиваться из-за padding и border.

==================================================
7. Стили html и body
==================================================

Правило:

html, body {
    height: 100%;
    margin: 0;
    font-family: ...;
    color: var(--text);
    background: ...;
    overflow: hidden;
}

Разбор свойств:

7.1. height: 100%

Позволяет html и body занимать всю высоту окна браузера.

Это нужно, чтобы контейнер `.wrap` мог корректно центрировать калькулятор по всей высоте экрана.

7.2. margin: 0

Убирает стандартные внешние отступы браузера у body.

Без этого по краям страницы часто остаются белые полосы.

7.3. font-family

Задает семейство шрифтов.

Здесь используется цепочка запасных вариантов:
- сначала Inter;
- потом ui-sans-serif;
- потом system-ui;
- затем другие системные шрифты.

Это делается для того, чтобы если один шрифт недоступен, браузер взял ближайший подходящий.

7.4. color

Задает основной цвет текста по умолчанию.

7.5. background

Здесь используется составной фон из нескольких слоев:
- radial-gradient(...)
- radial-gradient(...)
- linear-gradient(...)

Что это значит:
- `linear-gradient` создает плавный переход цвета по прямой;
- `radial-gradient` создает плавный переход от центра круга наружу.

В этом проекте фон состоит из:
- базового темного градиента;
- дополнительных световых пятен.

Так создается глубина и атмосферный techno/futuristic-эффект.

7.6. overflow: hidden

Скрывает содержимое, которое выходит за пределы видимой области.

Это полезно здесь, потому что есть декоративные фоновые эффекты и анимации, которые не должны вылезать наружу.

==================================================
8. Псевдоэлементы body::before и body::after
==================================================

Используются два псевдоэлемента:
- body::before
- body::after

Псевдоэлемент — это дополнительный визуальный слой, который создается CSS, без добавления нового HTML.

Общие свойства:

content: "";
position: fixed;
inset: 0;
pointer-events: none;

Разбор:

8.1. content: ""

Без свойства content псевдоэлемент обычно не появляется.

8.2. position: fixed

Фиксирует элемент относительно окна браузера.

Это значит, что декоративный слой остается привязанным ко всему экрану.

8.3. inset: 0

Краткая запись для:
- top: 0;
- right: 0;
- bottom: 0;
- left: 0;

То есть слой растягивается на весь экран.

8.4. pointer-events: none

Отключает реакцию на мышь.

Благодаря этому декоративные слои не мешают нажимать настоящие кнопки.

8.5. body::before

Этот слой создает сетку из линий через:
- background-image
- linear-gradient
- background-size
- mask-image
- opacity
- animation

Что значат свойства:

background-image — задает фоновое изображение. В данном случае вместо картинки используются градиенты.

background-size: 36px 36px — определяет размер ячейки фонового узора.

mask-image — маска видимости. Она делает так, что узор лучше виден в центре и затухает к краям.

opacity: 0.5 — задает прозрачность.

animation: gridMove 18s linear infinite — запускает анимацию:
- gridMove — имя анимации;
- 18s — длительность;
- linear — равномерная скорость;
- infinite — бесконечное повторение.

8.6. body::after

Этот слой добавляет размытые световые пятна.

Свойства:
- background — несколько radial-gradient;
- filter: blur(22px);
- opacity: 0.8;
- animation: floatGlow 10s ease-in-out infinite alternate;

Разбор:

filter: blur(22px) — размывает слой, делая свечение мягким.

ease-in-out — анимация ускоряется и замедляется плавно.

alternate — анимация идет то вперед, то назад, а не каждый раз начинается одинаково.

==================================================
9. Анимации через @keyframes
==================================================

В CSS есть:
- @keyframes gridMove
- @keyframes floatGlow
- @keyframes pulse

Что такое @keyframes:
- это описание анимации по этапам.

Пример:

@keyframes gridMove {
    from { transform: translateY(0); }
    to { transform: translateY(36px); }
}

Это значит, что элемент плавно смещается по вертикали.

Свойство transform используется для визуальных преобразований:
- translateY — движение по оси Y;
- translate3d — движение в 3D-координатах;
- scale — изменение размера.

==================================================
10. Контейнер .wrap
==================================================

.wrap {
    min-height: 100%;
    display: grid;
    place-items: center;
    padding: 24px;
    position: relative;
    z-index: 1;
}

Разбор свойств:

10.1. min-height: 100%

Минимальная высота контейнера равна высоте родителя. Это помогает занимать всю высоту экрана.

10.2. display: grid

Включает CSS Grid layout.

Grid — это система раскладки, которая позволяет удобно выравнивать элементы по строкам и колонкам.

10.3. place-items: center

Краткая запись, которая центрирует содержимое одновременно:
- по горизонтали;
- по вертикали.

То есть калькулятор оказывается в центре страницы.

10.4. padding: 24px

Добавляет внутренний отступ. Благодаря этому калькулятор не прилипает к краям экрана на маленьких устройствах.

10.5. position: relative

Делает контейнер опорной точкой для возможного абсолютного позиционирования внутренних элементов.

10.6. z-index: 1

Поднимает слой выше некоторых фоновых декоративных элементов.

==================================================
11. Основной блок .calculator
==================================================

.calculator {
    width: min(100%, 390px);
    padding: 20px;
    border-radius: calc(var(--radius) + 6px);
    background: ...;
    border: 1px solid var(--panel-border);
    box-shadow: var(--shadow);
    backdrop-filter: blur(18px);
    -webkit-backdrop-filter: blur(18px);
    position: relative;
    overflow: hidden;
}

Разбор:

11.1. width: min(100%, 390px)

Функция min() выбирает меньшее значение.

Это значит:
- блок не будет шире 390px;
- но на маленьком экране сможет занимать до 100% доступной ширины.

Так реализуется адаптивность.

11.2. padding: 20px

Внутренние отступы вокруг содержимого калькулятора.

11.3. border-radius

Скругляет углы.

Здесь используется:

calc(var(--radius) + 6px)

Функция calc() позволяет выполнять вычисления прямо в CSS.

11.4. background

Фон основного блока сделан градиентом, чтобы панель выглядела объемной.

11.5. border

Задает тонкую рамку вокруг блока.

11.6. box-shadow

Добавляет тень.

Тень делает элемент визуально отделенным от фона, как будто он находится выше поверхности.

11.7. backdrop-filter: blur(18px)

Размывает фон, который виден сквозь элемент.

Это создает эффект "стеклянной панели".

11.8. -webkit-backdrop-filter

Префиксированная версия для лучшей совместимости с браузерами на движке WebKit.

11.9. position: relative

Нужно, чтобы псевдоэлементы и абсолютные элементы внутри `.calculator` позиционировались относительно него.

11.10. overflow: hidden

Обрезает все декоративные слои, выходящие за пределы скругленной панели.

==================================================
12. Псевдоэлемент .calculator::before
==================================================

Этот псевдоэлемент создает дополнительную декоративную рамку с градиентным свечением.

Свойства:
- content: "";
- position: absolute;
- inset: -1px;
- border-radius: inherit;
- padding: 1px;
- background: linear-gradient(...);
- mask и mask-composite;
- pointer-events: none;

Разбор:

12.1. position: absolute

Абсолютное позиционирование располагает элемент внутри `.calculator`.

12.2. inset: -1px

Псевдоэлемент немного выходит за границы блока, чтобы рамка выглядела аккуратнее.

12.3. border-radius: inherit

Наследует скругление углов от родительского блока.

12.4. padding: 1px

Используется для построения тонкой обводки.

12.5. background: linear-gradient(...)

Градиент дает не однотонную, а светящуюся рамку.

12.6. mask / mask-composite

Это более сложная техника, которая позволяет оставить видимой только границу, а не весь слой целиком.

==================================================
13. Верхняя панель .topbar и ее элементы
==================================================

.topbar {
    display: flex;
    align-items: center;
    justify-content: space-between;
    gap: 12px;
    margin-bottom: 16px;
    position: relative;
    z-index: 1;
}

Что делает Flexbox здесь:
- display: flex — включает гибкую раскладку;
- align-items: center — выравнивает элементы по вертикали;
- justify-content: space-between — раздвигает первый элемент влево, второй вправо;
- gap: 12px — расстояние между элементами.

margin-bottom: 16px — отступ снизу от следующего блока.

`.brand` тоже использует flex, чтобы расположить значок и текст рядом.

`.brand-badge`:
- width / height задают размер;
- border-radius: 999px делает кружок;
- background задает цвет;
- box-shadow создает свечение;
- animation: pulse вызывает пульсацию.

`.brand-title`:
- font-size — размер шрифта;
- font-weight: 700 — жирность;
- letter-spacing — расстояние между буквами;

Правило `button { ... }` применяется ко всем кнопкам.

Разберем свойства подробно.

18.1. appearance: none

Убирает стандартный внешний вид кнопки, который навязывает браузер.

18.2. border: 0

Убирает стандартную рамку браузера.

18.3. cursor: pointer

Показывает курсор-указатель при наведении, сообщая, что элемент кликабелен.

18.4. border-radius: 18px

Скругляет углы кнопки.

18.5. padding: 16px 10px

Внутренние отступы:
- 16px сверху и снизу;
- 10px слева и справа.

18.6. font-size и font-weight

Управляют размером и насыщенностью текста.

18.7. color

Цвет текста кнопки.

18.8. background

Основной фон кнопки.

18.9. border

Тонкая полупрозрачная граница.

18.10. box-shadow

Внутренняя светлая линия через inset делает кнопку чуть более объемной.

18.11. transition

Позволяет плавно анимировать изменения свойств.

==================================================
19. Состояния кнопок: hover и active
==================================================

19.1. button:hover

Срабатывает, когда указатель мыши наведен на кнопку.

Изменения:
- transform: translateY(-2px);
- background становится ярче;
- border-color усиливается;
- box-shadow усиливается.

Эффект: кнопка будто слегка поднимается.

19.2. button:active и button.active

`:active` срабатывает при физическом нажатии.
`.active` добавляется JavaScript вручную для имитации нажатия с клавиатуры.

Изменения:
- transform: translateY(0) scale(0.98);
- фон меняется;
- появляется подсветка.

scale(0.98) — кнопка чуть уменьшается, создавая эффект продавливания.

==================================================
20. Псевдоэлемент button::before
==================================================

Он нужен для эффекта светового круга в точке касания.

Свойства:
- content: "";
- position: absolute;
- width: 0;
- height: 0;
- border-radius: 50%;
- background: radial-gradient(...);
- transform: translate(-50%, -50%);
- left: var(--x, 50%);
- top: var(--y, 50%);
- opacity: 0;
- z-index: -1;
- transition: ...

Разбор:

left и top используют CSS-переменные `--x` и `--y`, которые задает JavaScript при клике.

Это значит, что вспышка возникает именно в месте нажатия, а не просто в центре.

При active-состоянии:
- width становится 180px;
- height становится 180px;
- opacity становится 1.

То есть круг быстро увеличивается и создает эффект ripple/light pulse.

==================================================
24. Адаптивность через @media
==================================================

В конце файла есть:

@media (max-width: 420px) {
    ...
}

Это медиазапрос.

Он означает: "применять вложенные стили, если ширина экрана не больше 420px".

То есть эти правила работают на маленьких экранах, например на телефоне.

Что меняется:

24.1. `.calculator`
- меньше внутренние отступы;
- чуть меньше радиус скругления.

24.2. `.display`
- уменьшается минимальная высота;
- уменьшаются внутренние отступы.

24.3. `button`
- немного уменьшается padding;
- радиус становится чуть меньше.

24.4. `.tips`
- display: none;

Подсказки скрываются, чтобы не занимать лишнее место на маленьком экране.

25. Какие типы селекторов используются в CSS
==================================================

В этом проекте встречаются разные виды селекторов:

25.1. Селектор тега

Пример:
- button
- html
- body

Он выбирает все элементы данного тега.

25.2. Селектор класса

Пример:
- .wrap
- .calculator
- .display
- .operator

Он выбирает все элементы с указанным классом.

25.3. Групповой селектор

Пример:
- html, body
- body::before, body::after

Один стиль применяется сразу к нескольким селекторам.

25.4. Псевдоэлемент

Пример:
- ::before
- ::after

Добавляет декоративный виртуальный элемент.

25.5. Псевдокласс

Пример:
- :hover
- :active

Применяет стиль в определенном состоянии элемента.

25.6. Медиа-условие

Пример:
- @media (max-width: 420px)

Применяет стили только при определенных условиях экрана.

28. Как взаимодействуют HTML, CSS и JavaScript вместе
==================================================

В этом проекте они работают совместно:

HTML:
- создает структуру экрана;
- задает кнопки, блоки и подписи.

CSS:
- делает интерфейс красивым;
- задает расположение;
- управляет цветом, формой, анимацией и адаптивностью.

JavaScript:
- находит элементы по id и class;
- обрабатывает клики и клавиши;
- меняет текст в display;
- добавляет классы активного состояния;
- передает координаты клика в CSS-переменные для эффекта кнопки.

Особенно важно, что JavaScript и CSS здесь связаны через:
- класс `.active`;
- CSS-переменные `--x` и `--y`.

Это хороший пример того, как поведение и стиль работают вместе.
